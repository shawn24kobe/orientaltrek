# Google Search Console 操作检查清单

## 准备工作
- [ ] 确保网站已经上线并可访问
- [ ] 记下你的域名 (例如: foshan-furniture.com)
- [ ] 确保有Google账号
- [ ] 获取 FTP 或文件管理器访问权限 (用于验证)

---

## 第1部分: 在Google Search Console中注册

### 1.1 访问Google Search Console
- [ ] 打开浏览器，访问: https://search.google.com/search-console
- [ ] 点击 "开始使用"
- [ ] 用Google账号登录

### 1.2 添加资源
- [ ] 点击左上角的 "+"
- [ ] 选择 "添加资源"
- [ ] 选择 **"网域"** 选项 (推荐, 覆盖HTTP和HTTPS所有版本)
- [ ] 输入你的域名: `foshan-furniture.com` (不包含 https://)
- [ ] 点击 "继续"

---

## 第2部分: 验证域名所有权 (选择一个方法)

### 方法 A: DNS记录验证 (最推荐)
- [ ] 复制Google提供的TXT记录内容
- [ ] 登录你的域名注册商或托管商
- [ ] 进入DNS设置
- [ ] 添加新的TXT记录
- [ ] 粘贴Google提供的内容
- [ ] 保存更改 (可能需要24小时生效)
- [ ] 返回Google Search Console，点击 "验证"

### 方法 B: HTML文件验证
- [ ] 下载Google提供的HTML验证文件 (例如: google1234abcd.html)
- [ ] 将文件上传到网站根目录 (https://www.orientaltrek.com/google1234abcd.html)
- [ ] 验证文件可以访问
- [ ] 返回Google Search Console，点击 "验证"

### 方法 C: Google Analytics验证
- [ ] 如果已安装Google Analytics代码
- [ ] 选择此验证方法
- [ ] Google会自动验证

### 验证成功标志:
- [ ] 看到 "验证成功" 消息
- [ ] "所有权已验证" 标记出现
- [ ] 进入到主仪表板

---

## 第3部分: 提交Sitemap

### 3.1 导航到Sitemap页面
- [ ] 在左侧菜单找到 "Sitemaps"
- [ ] 点击进入

### 3.2 提交Sitemap
- [ ] 点击 "添加新的Sitemap"
- [ ] 输入: `sitemap.xml` (不需要完整URL)
- [ ] 点击 "提交"
- [ ] 等待处理提示

### 3.3 检查Sitemap状态
- [ ] 查看Sitemap列表
- [ ] 记录状态:
  - ✅ "成功" - 已接收并处理
  - ⏳ "待处理" - Google正在处理
  - ⚠️ "错误" - 需要修复

### Sitemap提交成功指标:
- [ ] 看到 "成功" 状态
- [ ] 显示 "已检索的URL数"
- [ ] 无错误消息

---

## 第4部分: 请求索引 (立即让Google爬取页面)

### 4.1 使用URL检查工具
- [ ] 点击左侧菜单 "URL检查"
- [ ] 在搜索框中输入第一个URL: `https://www.orientaltrek.com/`
- [ ] 按Enter

### 4.2 请求编写索引 - 首页
- [ ] 进入检查结果
- [ ] 如果显示 "URL在Google索引中" - 已索引 ✓
- [ ] 如果显示 "URL不在索引中" - 点击 "请求编写索引" 按钮
- [ ] 等待请求排队确认

### 4.3 为其他重要页面请求索引
按照相同流程为以下页面请求索引:

**必须提交的页面:**
- [ ] https://www.orientaltrek.com/products.html
- [ ] https://www.orientaltrek.com/about.html  
- [ ] https://www.orientaltrek.com/contact.html

**推荐提交的页面:**
- [ ] https://www.orientaltrek.com/collections.html
- [ ] https://www.orientaltrek.com/showroom.html
- [ ] https://www.orientaltrek.com/product-detail.html

---

## 第5部分: 监控覆盖率

### 5.1 查看覆盖率报告
- [ ] 点击左侧菜单 "覆盖率"
- [ ] 查看统计信息:
  - 有效页面数 - 已索引的页面
  - 仅含重定向 - 页面重定向
  - 错误 - 爬虫遇到的问题
  - 排除项 - 被排除的页面

### 5.2 解决错误
- [ ] 如果看到任何 "错误" 标记
- [ ] 点击展开查看具体问题
- [ ] 常见问题和修复:
  - **"网址无法访问"** - 检查服务器是否在线
  - **"404 Not Found"** - 检查URL是否正确
  - **"重定向错误"** - 检查重定向链
  - **"被 robots.txt 阻止"** - 检查robots.txt设置

### 5.3 性能监控
- [ ] 返回顶级报告
- [ ] 查看 "检查的URL总数"
- [ ] 查看 "不同的源错误"

---

## 第6部分: 设置搜索外观 (可选但推荐)

### 6.1 品牌验证
- [ ] 点击 "外观" → "品牌验证"
- [ ] 如果有公司标志，上传
- [ ] 填写品牌描述

### 6.2 添加网站链接搜索框
- [ ] 点击 "外观" → "搜索外观"
- [ ] 配置网站搜索框设置
- [ ] 这会让您的网站在搜索结果中显示搜索框

## 第7部分: 配置最佳做法

### 7.1 核心网页指标
- [ ] 点击 "体验" → "核心网页指标"

  - [ ] 重新测试

### 7.3 结构化数据
- [ ] 点击 "增强" → "结构化数据"
- [ ] 验证所有schema都被正确识别
- [ ] 查看是否有任何验证错误

## 故障排除

### Sitemap提交失败
**问题**: 看到 "错误" 状态
- [ ] 检查sitemap.xml是否可访问: https://www.orientaltrek.com/sitemap.xml
- [ ] 验证XML格式是否正确
- [ ] 确认所有URL都是有效的
- [ ] 等待24小时后重试

### URL请求索引无响应
**问题**: 请求已发送但没有更新
- [ ] 这是正常的 - Google会在后台处理
- [ ] 通常需要1-2周才能看到结果
- [ ] 继续监控覆盖率报告

### 页面不显示在索引中
**问题**: 几周后仍未被索引
- [ ] 检查页面内容质量
- [ ] 确保页面有足够的文字内容
- [ ] 检查是否有 noindex meta标签
- [ ] 验证页面可从其他页面链接
- [ ] 检查robots.txt是否阻止了该页面

### 出现大量错误
**问题**: 覆盖率显示多个错误
- [ ] 逐个点击错误类型查看详情
- [ ] 按URL查看具体问题
- [ ] 修复所有问题后重新请求索引
